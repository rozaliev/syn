AST->AST fold.

Each method of the Folder trait is a hook to be potentially overridden. Each method's default implementation recursively visits the substructure of the input via the noop_fold methods, which perform an "identity fold", that is, they return the same structure that they are given (for example the fold_crate method by default calls fold::noop_fold_crate).

If you want to ensure that your code handles every variant explicitly, you need to override each method and monitor future changes to Folder in case a new method with a new default implementation gets introduced.
